# CLAUDE.md — 開発ガイドライン

## プロジェクト概要

単一HTMLファイルで構成される個人用ToDoリストアプリ。サーバー不要、ブラウザ直接開き可能。

---

## ファイル構造

```
index.html    ← 唯一の実装ファイル（CSS・JSを内包）
.docs/spec.md → 仕様書
CLAUDE.md     → このファイル（開発ガイドライン）
```

---

## 実装ルール

- **単一ファイル原則**: すべてのHTML/CSS/JSは `index.html` に記述する。外部ファイル・ビルドツール・依存パッケージは一切使わない。
- **Vanilla JSのみ**: フレームワークやライブラリを使わず、純粋なJavaScriptで実装する。
- **LocalStorage永続化**: データはブラウザのlocalStorageに保存する。読み込み時は初期値のフォールバックも用意する。

---

## 開発指針

### コーディング規約
- JavaScript: ES2020以降、const/let使用、アロー関数OK
- CSS: セマンティックなクラス名、BEM風命名を参考に
- コメントは日本語で書く（仕様書と統一）

### テスト方針
- MVP段階ではブラウザでの手動動作確認を主とする
- 将来的にユニットテストを導入する場合は、単一HTMLのためPlaywrightやPuppeteerによるE2Eテストが適切

### 拡張性
- 仕様書 [.docs/spec.md](.docs/spec.md) に基づいて機能追加を行う
- v1（MVP）の範囲を超える変更は必ず仕様書を先に更新する
