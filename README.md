# ToDo リスト アプリ

個人が日常のToDoを管理するための軽量Webアプリ。サーバー不要で、ブラウザで直接開いて使えます。

---

## 使い方

1. [`index.html`](index.html) をブラウザで開く
2. タイトルを入力して「追加」ボタンをクリック
3. チェックボックスで完了/未完了を切り替え、✕ で削除

データを保存・読み込みするために `localStorage` を使用しています。リロードしてもデータは消えません。

---

## 機能一覧

| 機能 | 説明 |
|------|------|
| **ToDo の追加** | タイトル（必須）と詳細（任意）、期限日（任意）を設定して登録 |
| **一覧表示** | 新規 ToDo が最上部に表示。期限切れは視覚的に注意喚起 |
| **完了/未完了の切り替え** | チェックボックスでトグル。完了項目は取り消し線で区別 |
| **削除** | 各項目の「削除」ボタン（✕）で削除 |
| **インライン編集** | タイトル・詳細・期限日を直接クリックして編集。Enter で保存、Shift+Enter で改行 |
| **ドラッグ&ドロップ** | ドラッグハンドル（⋮⋮）で項目の順序を入れ替え |

---

## ファイル構成

```
index.html    ← すべてをここにまとめる（HTML + CSS + JS）
.docs/spec.md → 仕様書
tests/        ← Playwright E2E テスト
playwright.config.js
```

単一HTMLファイルで完結しており、外部依存・ビルドツールは一切使いません。

---

## データモデル (LocalStorage)

| キー | 値 |
|------|-----|
| `todos` | `{ version: number, items: TodoItem[] }` |

**TodoItem の構造:**

```jsonc
{
  "id": "一意ID",
  "title": "タイトル（必須）",
  "description": "詳細（任意）",
  "done": false,
  "dueDate": "2026-07-15",
  "createdAt": "2026-06-26T10:30:00Z"
}
```

## テスト

Playwright による E2E テストがあります。

```bash
# 全テストを実行
npx playwright test

# レポートをブラウザで表示
npx playwright show-report
```

| テストファイル | カバー範囲 |
|---------------|-----------|
| `tests/todo-basic.test.ts` | 追加・一覧表示・完了切替・削除・永続化 |
| `tests/todo-inline-edit.test.ts` | タイトル/詳細のインライン編集、バリデーション |
| `tests/todo-due-date.test.ts` | 期限日の設定・変更・削除 |
| `tests/todo-drag-drop.test.ts` | ドラッグ&ドロップによる並び替え |
