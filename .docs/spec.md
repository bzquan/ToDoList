# ToDo リスト アプリ — 仕様書

## 1. 概要

| 項目 | 内容 |
|------|------|
| **目的** | 個人が日常のToDoを管理するための軽量Webアプリ |
| **形式** | 単一HTMLファイル（`index.html`）— サーバー不要、ブラウザで直接開ける |
| **保存先** | ブラウザの `localStorage` — リロードしてもデータが消えない |

---

## 2. 機能一覧 (MVP)

### 2.1 ToDo の追加
- タイトル入力フィールドと「追加」ボタン
- タイトルは必須（空文字列は拒否、エラー表示）
- 詳細テキスト入力フィールドと「追加」ボタン
- 詳細は任意（空でも登録可能）
- 登録時刻を自動記録（内部管理用）
- 期限日付（任意）を設定できる入力欄を用意

### 2.2 ToDo の一覧表示
- 新規ToDoが最上部に追加される
- 各ToDoに表示名・完了状態・期限日・登録時刻を表示
- タイトルと詳細の両方を含む本文を表示
- 期限切れのToDoは視覚的に注意喚起（色など）
- データがない場合は「ToDoがありません」と表示

### 2.3 完了/未完了の切り替え
- チェックボックスまたはクリックで完了状態をトグル
- 完了済みの項目は取り消し線などで視覚的に区別

### 2.4 ToDo の削除
- 各項目に「削除」ボタン（✕）
- 削除確認ダイアログは省略（MVPのため）

### 2.5 期限日付の変更・設定
- 期限日付が設定済みの項目：期限テキストをクリックするとインラインで `type="date"` の入力欄に切り替わる
- 期限日付が未設定の項目：「📅 期限設定」リンクが表示され、クリックで同様にインライン入力欄に切り替わる
- 入力中にフォーカスが外れるか Enter キーが押されると、新しい期限日付として保存・反映される
- 空にすることで期限日付を削除できる
- 完了済みの項目でも期限日付の変更は可能

### 2.6 ドラッグ&ドロップで並び替え
- ToDo 項目の左端にあるドラッグハンドル（⋮⋮）をドラッグして、他の項目との間でドロップすることで順序を入れ替えられる
- ドロップ先の位置に応じて `items` の配列順が更新され、localStorage に保存される
- ドラッグ中は視覚的に移動中であることを示す（半透明化またはプレースホルダーの表示）

## 3. データモデル (LocalStorage)

```jsonc
// key: "todos"
{
  "version": 1,                    // スキーマバージョン（将来の互換性用）
  "items": [
    {
      "id": "uuid_or_timestamp",             // 一意ID
      "title": "買い物に行く",                  // タイトル（必須）
      "description": "牛乳と卵を買う",          // 詳細（任意）
      "done": false,                          // 完了状態
      "dueDate": "2026-07-15",                // 期限日（YYYY-MM-DD、任意）
      "createdAt": "2026-06-26T10:30:00Z"     // ISO形式の登録時刻
    }
  ]
}
```

## 4. UI/UX ガイドライン

| 要素 | ガイドライン |
|------|-------------|
| **レイアウト** | シンプル中央寄せ。ヘッダー → 入力フォーム → リスト |
| **レスポンシブ** | スマホでも使えるよう、幅は `max-width: 600px` で中央寄せ |
| **アクセシビリティ** | ラベル付き input、ボタンに `aria-label` |
| **スタイル** | CSS は `<style>` タグ内に記述。外部ファイル依存なし |

---

## 5. ファイル構成

```
index.html    ← すべてをここにまとめる
├── <head>
│   ├── meta charset, viewport
│   └── <style>     (CSS)
├── <body>
│   ├── ヘッダー
│   ├── 入力フォーム (<form>)
│   └── ToDo リスト (<ul> or <div>)
└── <script>      (JS: localStorage CRUD)
```

---

## 6. 実装の優先順位

| 優先度 | 内容 |
|--------|------|
| **P0** | データモデル + CRUD（追加・一覧・完了切替・削除） |
| **P1** | レスポンシブデザイン + アクセシビリティ |
| **P2** | エラーハンドリング（空文字拒否など） |
