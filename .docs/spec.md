# ToDo リスト アプリ — 仕様書

## 1. 概要

| 項目 | 内容 |
|------|------|
| **目的** | 個人が日常のToDoを管理するための軽量Webアプリ |
| **形式** | 単一HTMLファイル（`index.html`）— サーバー不要、ブラウザで直接開ける |
| **保存先** | ブラウザの `localStorage` — リロードしてもデータが消えない |

---

## 2. 機能一覧 (MVP)

### 2.1 ToDo の追加
- テキスト入力フィールドと「追加」ボタン
- 空文字列は拒否（エラー表示）
- 登録時刻を自動記録（内部管理用）

### 2.2 ToDo の一覧表示
- 新規ToDoが最上部に追加される
- 各ToDoに表示名・完了状態・登録時刻を表示
- データがない場合は「ToDoがありません」と表示

### 2.3 完了/未完了の切り替え
- チェックボックスまたはクリックで完了状態をトグル
- 完了済みの項目は取り消し線などで視覚的に区別

### 2.4 ToDo の削除
- 各項目に「削除」ボタン（✕）
- 削除確認ダイアログは省略（MVPのため）

---

## 3. データモデル (LocalStorage)

```jsonc
// key: "todos"
{
  "version": 1,                    // スキーマバージョン（将来の互換性用）
  "items": [
    {
      "id": "uuid_or_timestamp",   // 一意ID
      "text": "買い物に行く",        // ToDo本文
      "done": false,                // 完了状態
      "createdAt": "2026-06-26T10:30:00Z"  // ISO形式の登録時刻
    }
  ]
}
```

---

## 4. UI/UX ガイドライン

| 要素 | ガイドライン |
|------|-------------|
| **レイアウト** | シンプル中央寄せ。ヘッダー → 入力フォーム → リスト |
| **レスポンシブ** | スマホでも使えるよう、幅は `max-width: 600px` で中央寄せ |
| **アクセシビリティ** | ラベル付き input、ボタンに `aria-label` |
| **スタイル** | CSS は `<style>` タグ内に記述。外部ファイル依存なし |

---

## 5. ファイル構成

```
index.html    ← すべてをここにまとめる
├── <head>
│   ├── meta charset, viewport
│   └── <style>     (CSS)
├── <body>
│   ├── ヘッダー
│   ├── 入力フォーム (<form>)
│   └── ToDo リスト (<ul> or <div>)
└── <script>      (JS: localStorage CRUD)
```

---

## 6. 実装の優先順位

| 優先度 | 内容 |
|--------|------|
| **P0** | データモデル + CRUD（追加・一覧・完了切替・削除） |
| **P1** | レスポンシブデザイン + アクセシビリティ |
| **P2** | エラーハンドリング（空文字拒否など） |
